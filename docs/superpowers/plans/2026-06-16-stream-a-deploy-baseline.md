# Stream A — Deploy-baseline & CI-gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Gøre ChainHubs deploy-vej og CI-gate reel: én baseline Prisma-migration der rejser hele skemaet mod en tom DB, en virkende lint-kommando der fanger de 4 reelle ESLint-errors, en fungerende prod-cron, og docs der ikke vildleder fremtidige agenter.

**Architecture:** Squash de 7 eksisterende AI-incremental-migrationer til ét `0_init` genereret `--from-empty → schema.prisma`. Skift `next lint` (ugyldig i Next 16) til `eslint src`, ret de 4 errors den afslører, tilføj `vercel.json` med crons-blok, bump CI-Node, og synkronisér `CLAUDE.md` m.fl. med virkeligheden.

**Tech Stack:** Prisma 6 (migrate diff/deploy/reset), Next.js 16, ESLint 9 (flat config), Prettier 3, Vercel Cron, lokal Docker-Postgres som scratch-DB.

**Forudsætning:** Ingen prod-DB eksisterer endnu (go-live ikke sket) → squash er sikkert; dev-DB kan nulstilles/reseedes.

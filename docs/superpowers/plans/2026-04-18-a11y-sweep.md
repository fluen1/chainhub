# ChainHub A11y Sweep Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring ChainHub til WCAG 2.1 Level A compliance plus kritisk Level AA (kontrast, tastatur-nav), ved at enable `eslint-plugin-jsx-a11y`, retrofit forms med `htmlFor/id`-parring, bygge en genanvendelig `AccessibleDialog`-primitive med focus-trap/Escape/aria-labelledby, tilføje skip-to-main-link, og sikre tastatur-adgang til kanban-drag-drop.

**Architecture:** En primitive-først tilgang: byg `AccessibleDialog` én gang, refaktorér 3 eksisterende modals til at bruge den. Form-retrofit følger eksisterende `UploadVersionForm`-mønster (godt htmlFor/id). Kanban får keyboard-fallback via arrow-keys på kort. ESLint-plugin fanger fremtidige regressioner.

**Tech Stack:** React 18 (useRef, useEffect for focus management), Tailwind (focus styles), eslint-plugin-jsx-a11y, ingen nye dependencies.

---

## Kontekst

A11y-audit (2026-04-18) identificerede:

- **1 aria-attribut** i hele kodebasen (bortset fra 2 roles)
- **7 forms** med label/input uden htmlFor/id (CreateTaskForm, CreateVisitForm m.fl.)
- **3 modals** uden focus-trap, aria-labelledby eller Escape-close
- **Kanban drag-drop** uden keyboard-fallback
- **ESLint-plugin installed men ikke enabled** — ingen static checks
- **0 skip-to-main links** — keyboard-first brugere kan ikke springe nav
- **gray-400/500 helper text** — potentielt fail på WCAG AA kontrast

UploadVersionForm bruger allerede korrekt htmlFor/id-mønster → dette er template.

**Udfald:** Alle forms a11y-compliant, alle modals har focus-trap, kanban har keyboard-fallback, ESLint fanger regressioner. Audit-score fra ~2/10 → ~8/10.

### Task 4: Byg AccessibleDialog-primitive

**Files:** Create `src/components/ui/accessible-dialog.tsx`

- [ ] **Step 1: Opret filen med fuld implementation**

```tsx
'use client'

import { useEffect, useRef, type ReactNode } from 'react'
import { cn } from '@/lib/utils'

interface AccessibleDialogProps {
  open: boolean
  onClose: () => void
  title: string
  titleId?: string
  children: ReactNode
  className?: string
}

const FOCUSABLE_SELECTOR =
  'a[href], button:not([disabled]), textarea:not([disabled]), input:not([disabled]), select:not([disabled]), [tabindex]:not([tabindex="-1"])'

export function AccessibleDialog({
  open,
  onClose,
  title,
  titleId = 'dialog-title',
  children,
  className,
}: AccessibleDialogProps) {
  const dialogRef = useRef<HTMLDivElement>(null)
  const previouslyFocused = useRef<HTMLElement | null>(null)

  // Luk på Escape
  useEffect(() => {
    if (!open) return
    const handler = (e: KeyboardEvent) => {
      if (e.key === 'Escape') onClose()
    }
    document.addEventListener('keydown', handler)
    return () => document.removeEventListener('keydown', handler)
  }, [open, onClose])

  // Focus-trap + focus-restore
  useEffect(() => {
    if (!open) return
    previouslyFocused.current = document.activeElement as HTMLElement | null

    // Flyt focus til første fokuserbare element i dialogen
    const first = dialogRef.current?.querySelectorAll<HTMLElement>(FOCUSABLE_SELECTOR)[0]
    first?.focus()

    const trap = (e: KeyboardEvent) => {
      if (e.key !== 'Tab' || !dialogRef.current) return
      const focusables = dialogRef.current.querySelectorAll<HTMLElement>(FOCUSABLE_SELECTOR)
      if (focusables.length === 0) return
      const firstEl = focusables[0]!
      const lastEl = focusables[focusables.length - 1]!
      if (e.shiftKey && document.activeElement === firstEl) {
        e.preventDefault()
        lastEl.focus()
      } else if (!e.shiftKey && document.activeElement === lastEl) {
        e.preventDefault()
        firstEl.focus()
      }
    }
    document.addEventListener('keydown', trap)
    return () => {
      document.removeEventListener('keydown', trap)
      previouslyFocused.current?.focus()
    }
  }, [open])

  if (!open) return null

  return (
    <div
      className="fixed inset-0 z-50 flex items-center justify-center bg-black/40 p-4"
      onClick={onClose}
    >
      <div
        ref={dialogRef}
        role="dialog"
        aria-modal="true"
        aria-labelledby={titleId}
        className={cn('bg-white rounded-lg shadow-xl max-w-lg w-full p-6', className)}
        onClick={(e) => e.stopPropagation()}
      >
        <h2 id={titleId} className="text-lg font-semibold text-gray-900 mb-4">
          {title}
        </h2>
        {children}
      </div>
    </div>
  )
}
```

import { describe, it, expect, vi } from 'vitest'
import { render, screen, fireEvent } from '@testing-library/react'
import { AccessibleDialog } from '@/components/ui/accessible-dialog'

describe('AccessibleDialog', () => {

```tsx
const [grabbedTaskId, setGrabbedTaskId] = useState<string | null>(null)

function handleCardKeyDown(e: React.KeyboardEvent, taskId: string, currentStatus: string) {
  if (e.key === ' ' || e.key === 'Enter') {
    e.preventDefault()
    setGrabbedTaskId(grabbedTaskId === taskId ? null : taskId)
  }
  if (grabbedTaskId === taskId && (e.key === 'ArrowLeft' || e.key === 'ArrowRight')) {
    e.preventDefault()
    const statuses = ['NY', 'AKTIV_TASK', 'AFVENTER', 'LUKKET']
    const idx = statuses.indexOf(currentStatus)
    const nextIdx = e.key === 'ArrowRight' ? idx + 1 : idx - 1
    if (nextIdx >= 0 && nextIdx < statuses.length) {
      // Kald eksisterende status-update-handler
      moveTask(taskId, statuses[nextIdx] as never)
    }
  }
}
```

- [ ] **Step 2: Skift text-gray-400 → text-gray-500 i brødtekst-kontekster**

`text-gray-400` på hvid baggrund = ~3.1:1 kontrast → fail WCAG AA (krav 4.5:1).
`text-gray-500` = ~4.6:1 → pass AA.
`text-gray-600` = ~7:1 → pass AAA.

**Regel:**

- Hjælpe-/beskrivelsestekst → `text-gray-500` minimum
- Placeholder/disabled → `text-gray-400` acceptabelt (ikke brødtekst)
- Vigtig tekst (labels, værdier) → `text-gray-700` eller mørkere

Brug `grep` + manuelle check pr. fil. Ignore disabled-states og icon-farver.

- [ ] **Step 3: Dokumentér baseline**

- Hjælpetekst: `text-gray-500` minimum (ikke gray-400)

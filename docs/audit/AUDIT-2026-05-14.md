# ChainHub — Komplet gennemgang 2026-05-14

**Metode:** Statisk kortlægning via 6 parallelle Explore-agents på tværs af alle 24 dashboard-routes, 26 server actions og 9 API-routes. Hybrid med planlagt Playwright-validering på top-fund (Fase 2 herunder).

**Status:** B-stil overhaul færdig (commit `7b5f622`), 697 tests grønne, 41+ routes i build. Men auditet afdækker **38 kritiske, ~55 vigtige og ~30 kosmetiske fund** — flere er reelle data-leaks eller helt manglende UI til actions der allerede er implementeret.

**Fase 2 status:** Playwright-walkthrough kørt på dev-server port 3010, login som GROUP_OWNER + GROUP_FINANCE. **Alle 19 spotchecked fund blev live-bekræftet.** 3 nye fund afdækket runtime (NF-1 til NF-3 nedenfor). Detaljer i `AUDIT-PLAYWRIGHT-2026-05-14.md`.

**Tre overordnede mønstre:**

1. **RBAC-regressioner efter B-stil port** — flere queries og UI-gates mangler scope-filter (`organization_id` / `company_id: { in: companyIds }`)
2. **Døde knapper** — minimum 19 knapper i UI'et linker til ikke-eksisterende routes eller har ingen `onClick`/`href`
3. **Orphaned features** — implementeret server-side men ikke wired til UI: GDPR Article 15+17 panel, ContractStatusForm, deleteDocument, updateTask{Priority,Assignee,DueDate}

## 🔴 KRITISK — nye fund kun afdækket runtime

Disse tre var ikke i den statiske audit fordi de først blev synlige ved live-navigation.

### NF-1 — `NEXTAUTH_URL` hardkodet til port 3000

`.env.local` har `NEXTAUTH_URL=http://localhost:3000`. Signout via `/api/auth/signout` redirecter brugeren til en helt anden applikation på port 3000. Brydes også ved deploy hvis env-var ikke korrekt sættes. **Fix:** brug `NEXTAUTH_URL=http://localhost:${PORT}` med dynamic port, eller dokumentér env-required at matche dev-port.

### NF-2 — `/documents/upload` route eksisterer ikke

"↑ Upload dokument"-knappen på `/documents` → 404. Knappen er primær upload-entry på dokumentlisten. **Fix:** opret `src/app/(dashboard)/documents/upload/page.tsx` eller refaktorer til modal-flow.

### NF-3 — `/companies/[id]/stamdata` route eksisterer ikke

"Rediger stamdata"-knappen på enhver selskab-detail-side → 404. Var antaget i statisk audit at sub-routen eksisterede; gør den ikke. **Fix:** Konvertér til `EditStamdataDialog`-modal (matcher memory-beskrivelsen) eller opret route.

5. **`src/lib/permissions/index.ts:149`** — `canAccessModule` for `'persons'` og `'documents'` returnerer altid `true` (default-case). Alle autentificerede brugere kan se alle personer/dokumenter uanset rolle.

7. **`src/app/(dashboard)/companies/page.tsx:92`** — raw SQL bruger `status = 'UDLOBET'` (uden E). Korrekt enum-værdi: `UDLOEBET`. Query matcher 0 rækker → health-beregning markerer aldrig selskaber som critical pga. udløbne kontrakter.

9. **`src/actions/company-detail.ts:191-200`** — Finance hardcodet til `period_year: 2025` og `2024`. Ved kalenderårsskift viser FinanceSection `null` for alle selskaber.

### Døde flows / stubs i produktion

Spec siger leveret — koden er ren `<button onClick>`. Plus PROGRESS.md angiver det som ✅ leveret.
13. **`src/app/(dashboard)/persons/[id]/person-detail-b.tsx`** — GDPR-panel eksisterer slet ikke. `prepareGdprExport` + `executeGdprDelete` actions er implementerede og testede, men der er ingen knapper, ingen navn-typing-validering, ingen UI-overflade. Article 15 (export) og 17 (erasure) er kun tilgængelige via direkte API-URL.
14. **`src/app/(dashboard)/settings/settings-b.tsx:124`** — AI-usage progress-bar hardcodet til `{used: 0, max: 1000, percent: 0}`. Reelt forbrug usynligt i /settings (men korrekt på /settings/ai-usage).

## Tværgående mønstre

### Mønster 1: B-stil overhaul har efterladt 6 forms uporterede

Alle `Create*Form.tsx` (Company, Case, Contract, Task, Visit, Person) bruger stadig plain Tailwind med `bg-blue-600` submit-knapper og `border-gray-300` inputs. /visits/[id] hele siden ditto. Visuel inkonsistens på alle "opret"-flows.

### Mønster 2: "Rediger"-routes eksisterer ikke nogen steder

Detail-sider for kontrakter, sager, opgaver og personer har alle "Rediger"-knapper der linker til `/{modul}/{id}/edit`. Ingen af disse routes findes. Total: minimum 8 døde redigeringsknapper på tværs af appen.

### Mønster 3: Implementerede actions uden UI

- `ContractStatusForm` → ikke importeret nogen steder på B-stil
- `deleteDocument` → ingen UI-knap
- `updateTaskPriority`, `updateTaskAssignee`, `updateTaskDueDate` → kun edit-flow, men edit-route eksisterer ikke
- `prepareGdprExport`, `executeGdprDelete` → ingen UI på persons-detail
- `updateOrganization` → fungerer, men nogen felter (`last_login_at`) hentes ikke

### Mønster 4: RBAC-scope mangler i 6+ queries

Adresseret per-fil under KRITISK-listen. Pattern: `where: { organization_id }` uden `company_id: { in: companyIds }`. Konsekvens: cross-company data lækker for READONLY-roller.

### Mønster 5: Hardcoded enum-værdier som dansk tekst i actions/views

Bør konsolideres: ingen `getXxxLabel()`-funktion nedenfor `src/actions/` eller `src/app/` skal returnere danske strings. `labels.ts` er kanonisk.
